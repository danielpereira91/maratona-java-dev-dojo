Interfaces

public interface NomeDaInterface

Parte 01 - Aula 87

Cria-se uma Interface clicando com o botão direito no package desejado (ou alt + insert), clicando em "New", depois "Java Class" e selecionando Interface.

Todos os métodos criados dentro de uma interface são pulic e abstract, então não há necessidade de declarar.

Quando trabalhando com Interfaces, não utiliza-se extends, mas implements, e o nome da interface.
public class NomeDaClasse implements NomeDaInterface

Parte 02 - Aula 88

Não existe limite para a quantidade de interfaces que você pode implementar em uma classe.
public class NomeDaClasse implements NomeDaInterface1, NomeDaInterface2

A partir do Java 8 foi possível adicionar métodos com funcionalidades nas Interfaces, bastando adicionar "default" antes de void.
default void nomeDoMétodo

Parte 03 - Aula 89

As Interfaces são uma espécie de contrato onde todos os métodos, por padrão, serão públicos e abstratos.
Se não criar como abstratos, eles precisam ser default (introduzido no Java 8), e é necessário prover uma implementação.

Não é possível instanciar um objeto através de uma interface.

Ordem dos modificadores de acesso

Mais restritivo ao menos restritivo

private
default
protected
public

Os modificadores de acesso só pordem ser sobrescritos por modificadores mais liberais

Uma classe private pode ser sobrescrita com todos os outros modificadores, já o public não pode ser sobrescrito por nenhum outro.
