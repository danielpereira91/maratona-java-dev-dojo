Classes Abstratas

Parte 01

As classes abstratas existem para servir como template para classes extendidas.

São um tipo de classe especial que não pode ser instanciada, apenas herdada. Sendo assim, uma classe abstrata não pode ter um objeto criado a partir de sua instanciação. Essas classes são muito importantes quando não queremos criar um objeto a partir de uma classe “geral”, apenas de suas “subclasses”.

Parte 02

Classes Abstratas podem conter médtodos abstratos e não-abstratos. Mas classes concretas não podem conter métodos abstratos.

Torna-se útil ter um método para, por exemplo, excluir um funcionário do sistema. Você pode ter o método concreto dentro da classe abstrata Funcionario.

Métodos abstratos não têm corpo. A partir do momento em que você cria o método abstrato na superclasse, obrigatoriamente você precisa também implementá-lo nas subclasses, mesmo que sem corpo também.

Parte 03

Caso haja mais classes abstratas parentes (como classe avô etc) e nelas houverem métodos abstratos, as subclasses abstratas abaixo não precisarão implementar o método abstrato, somente as classes concretas que deverão fazer a implementação.

Se a classe abstrata implementar o método, as classes filhas dessa classe já pegarão este método.
